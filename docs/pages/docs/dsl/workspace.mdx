# Workspace

Workspace is a directory with LikeC4 files, that are merged into single model.  
You split and organize the workspace as you want.

## Example

<FileTree>
  <FileTree.Folder name="cloud" open>
    <FileTree.File name="service1.c4" />
    <FileTree.File name="service2.c4" />
  </FileTree.Folder>
  <FileTree.Folder name="externals" open>
    <FileTree.File name="amazon.c4" />

  <FileTree.File name="landscape.c4" />
  <FileTree.File name="specs.c4" />

<Tabs items={['specs.c4', 'landscape.c4', 'cloud/service1.c4', 'externals/amazon.c4']}>
  <Tab>
    This file defines only the specification.

    ```likec4    
    specification {
      element actor {
        style {
          shape person
        }
      }
      element system
      element service
      element externals
    }
    ```  
  </Tab>
  <Tab>
    This file describes only top-level elements.

    ```likec4  
    model {
      customer = actor 'Customer'
      cloud = system 'Cloud'

      customer -> cloud 'uses cloud services'
    }  
    ```  
  </Tab>
  <Tab>
    This file _extends_ the model of `cloud` and adds `service1`.  
    We use separate file to describe the internals of `cloud`.

    ```likec4    
    model {
      extend cloud {
        service1 = service 'Service 1'
      }
    }
    ```  
  </Tab>
  <Tab>
    Here we are referencing the `cloud.service1` defined at `cloud/service1.c4`.

    ```likec4    
    model {
      amazon = externals 'Amazon Web Services' {
        rds = service 'Database'
      }

      cloud.service1 -> amazon.rds
    }
    ```  
  </Tab>
</Tabs>
