# Specification

In the `specification` section you customize your model.

## Element kind [#kind]

The `element` defines the kind that can be used across the model.

```likec4 copy
specification {

  // Define whatever you want
  element user
  element cloud
  element system
  element application
  element component
  element controller
  element microservice
  element queue
  element restapi
  element graphqlMutation
  element repository
  element database
  element pgTable

}
```

The definition may have a nested `style` block with properties.
These properties are defaults for all elements of this kind.

```likec4 copy
specification {

  element user {
    style {
      shape person
    }
  }

  element spa {
    style {
      shape: browser // ':' is optional, but if you prefer
      color: amber
    }
  }

  element queue {
    style {
      shape queue
      color gray
    }
  }

}
```

### Shapes

Available element shapes:

| shape     | example |
| :-------- | :-----: |
| rectangle |   ..    |
| storage   |   ..    |
| cylinder  |   ..    |
| browser   |   ..    |
| mobile    |   ..    |
| person    |   ..    |
| queue     |   ..    |

If not set, the default shape is `rectangle`.

### Icon

Element kind may have an icon: any image, publicly available with `https://`.  
Some examples - [https://icons.terrastruct.com/](https://icons.terrastruct.com/)

```likec4 copy
specification {

  element k8sService {
    style {
      icon https://icons.terrastruct.com/azure%2F_Companies%2FKubernetes.svg
    }
  }

}
```

## Tag

Tags may be used to mark, group, filter elements/relationships/views, or give some additional semantics, like `#deprecated`, `#epic123` or `#team2`.

```likec4
specification {

  tag deprecated
  tag epic123
  tag team2

}
```
