## Specification

In the `specification` section you customize your model.

### Element kind [#kind]

The `element` defines the kind that can be used across the model.

```likec4 copy
specification {

  // Define whatever you want
  element user
  element cloud
  element system
  element application
  element component
  element controller
  element microservice
  element queue
  element restapi
  element graphqlMutation
  element repository
  element database
  element pgTable

}
```

Element kinds are required for the [model](./model).

### Tag

Tags may be used to mark, group, filter elements/relationships/views, or give some additional semantics, like `#deprecated`, `#epic123` or `#team2`.

```likec4
specification {

  tag deprecated
  tag epic123
  tag team2

}
```
### Relationships

Relationships can be used to define the kind and styling of a relation (`-[<name>]->`).
```likec4
specification {
  relationship async {
    color amber
    line dotted
    head: diamond
    tail: diamond
  }
}
```

#### Color
Any color defined in [styling](./styling).

#### Line
Valid options for `line` are: `solid`, `dashed`, `dotted`.

#### Arrowhead / Arrowtail
Valid options for `head` and `tail` are: `normal`, `onormal`, `diamond`, `odiamond`, `vee`, `none`.
Refer to [Graphviz arrowType](https://graphviz.org/docs/attr-types/arrowType/) for visuals.
